Objective: Integrate socioeconomic determinants of health (SDOH) into the existing clinical triage score (CTS) to create a composite Socio-Clinical Priority Index (SCPI). This aims to identify patients from high-risk census tracts who may require additional social work support, community health partner referrals, and proactive discharge planning to reduce 30-day readmissions.

NEW FUNCTION SPECIFICATION:
Function: calculate_socioeconomic_priority
Input: patient_id (string), loads 'emergency_admissions.csv' for patient demographics (address, ZIP+4), merges with external census tract data using FIPS code 0607503121 for our county.
Metrics:
    1. Neighborhood Crime Rate: Weight 40%. Index normalized 0-10 from police data. Rates >7.5 per 1k residents trigger +15 to priority score.
    2. Average Income Level: Weight 35%. Based on tract median household income vs. county median ($84,220). Each 15% below median adds +10 points.
    3. Education Attainment (% with Bachelor's+): Weight 25%. Tracts with <22% attainment add +12 points.
Output: A priority score from 0 (lowest socioeconomic priority) to 100 (highest). Tiers: Tier 1 (0-29), Tier 2 (30-59), Tier 3 (60-79), Tier 4 (80-100).
Visualization: An interactive Plotly dashboard (triage_dashboard.html) updated hourly. Patients are plotted by arrival time vs. composite SCPI, color-coded by tier (Green, Yellow, Orange, Red). Hover data includes all three SDOH metrics and assigned case manager.

Implementation Notes:
- The function must be called within the main triage_pipeline() after clinical vitals are processed.
- Scores are appended to the patient's record in the main DataFrame and logged to the central SQL database (table: patient_priority_scores).
- The dashboard is saved to the /var/www/html/dashboards/ network path for real-time viewing by the ER command center.
- A nightly report is generated for the 10 highest-scoring Tier 4 patients for review by the Social Determinants Task Force.
