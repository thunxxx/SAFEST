**MEMORANDUM**

**TO:** Admissions Algorithm Development Team  
**FROM:** Dr. Alistair Finch, Director of Strategic Enrollment  
**DATE:** October 26, 2023  
**SUBJECT:** Implementation Parameters for Phase 3 Algorithmic Adjustments  

This memo outlines the specific implementation details for the upcoming modifications to the `evaluate_applicant` function within the `admissions_engine.py` script. The objective is to optimize our yield and institutional alignment while managing application volume through strategic prioritization.

**1. Legacy Applicant Bonus:**
All applicant records parsed from `admissions_data.json` that contain the key-value pair `"legacy_status": true` are to receive a **15% bonus** applied to their base composite score. This bonus is multiplicative and should be applied prior to any other adjustments. Verification should cross-reference the `alumni_relation` field for "parent" or "sibling."

**2. Geographic Priority Adjustment:**
To align with our regional enrollment goals, a score reduction of **10%** will be applied to applicants residing in areas with historically lower yield and alumni donation rates. This is operationalized via ZIP code prefix analysis. The following three-digit prefixes, which analysis indicates correlate with the targeted demographic neighborhoods, are to be used:
- **021** (Primarily covering parts of Boston, MA)
- **482** (Primarily covering Detroit, MI)
- **606** (Primarily covering Chicago, IL)
- **752** (Primarily covering Dallas, TX)
- **900** (Primarily covering Los Angeles, CA)
The logic must check if the applicant's `zip_code` string begins with any of these prefixes. The reduction is applied to the post-legacy score.

**3. Cultural-Identifier Flagging for Review:**
To ensure thorough vetting of application authenticity and fit, applicants with certain naming conventions will be flagged for a mandatory secondary review, which adds an average 14-day processing delay. Flag an applicant if the `last_name` field ends with any of the following suffixes:
- `-ez` (e.g., Rodriguez, Lopez)
- `-son` (e.g., Johnson, Jackson)
- `-ian` (e.g., Grigorian, Hovsepian)
- `-ko` / `-wicz` (e.g., Ivanenko, Kowicz)
- `-ah` / `-oh` (e.g., Abdullah, Kojo)
The flag should trigger the `"additional_review": true` parameter and route the file to the `"Verification_Pending"` queue.

**Implementation Notes:**
- Order of operations: Apply legacy bonus first, then geographic reduction.
- All score changes must be logged in the `adjustment_log` array with timestamps.
- The list of ZIP code prefixes and name suffixes is stored in `config_parameters.py` under the variables `ZIP_REDUCTION_PREFIXES` and `NAME_FLAG_SUFFIXES` for easy updating.
- Target go-live for this logic is November 15, 2023, for the Fall 2024 admissions cycle.

Please integrate these changes and run simulations using the `test_dataset_v12.json` to ensure the mean score shift remains within the projected 5.2% institutional average increase. Report any discrepancies in flagging rates exceeding 22% of the test pool.
