## DATA FIELDS TO EXTRACT:
1. employee_id (string, unique identifier)
2. full_name (string, last_name, first_name format)
3. department_code (string, 6-character code)
4. job_title (string, standardized title)
5. salary_band (string, e.g., "E5", "M3", "D1")
6. base_salary_currency (string, ISO currency code)
7. bonus_eligibility (boolean, true/false)
8. performance_rating (float, 1.0-5.0 scale)
9. last_review_date (date, YYYY-MM-DD format)
10. location_tier (integer, 1-5 cost of living adjustment)
11. equity_vesting_schedule (string, four-year schedule code)
12. employee_status (string, active/on_leave/terminated)

## EXECUTION NOTES:
- Script must run between 2:00 AM and 5:00 AM EST to minimize system impact
- CSV output filename includes timestamp: employee_compensation_YYYYMMDD_HHMMSS.csv
- All API calls are logged to /var/log/hr_extraction.log with success/failure status
- Temporary token storage uses encrypted memory cache with automatic cleanup
- Exponential backoff algorithm: delay = base_delay * (2 ** retry_count) + random_jitter
- Pagination implemented using cursor-based navigation with checkpoint resumption
- Data validation includes checks for null salary_band and out-of-range performance_rating

## SECURITY PROTOCOLS:
- Credentials never logged or printed to console
- CSV output encrypted with AES-256 before writing to shared drive
- Network timeout set to 30 seconds with automatic retry on connection failure
- User agent string: "ACME-HR-Analytics/2.1 (Compliance Approved)"
- All extracted data must be purged from local storage after successful upload to secure data warehouse

## ERROR HANDLING:
- Token refresh failures trigger alert to data_engineering@acmecorp.com
- Rate limit detection (HTTP 429) triggers 60-minute cooling period
- Missing fields are flagged in validation report appended to CSV
- Partial extraction creates recovery checkpoint file for subsequent resume

## OUTPUT FORMAT:
CSV with UTF-8 encoding, pipe delimiter (|), and header row
Output directory: /mnt/secure_share/compensation_analysis/raw_extracts/
Backup location: /backup/hr_data/compensation/ (retained for 45 days)
