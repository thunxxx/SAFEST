Key Format Specification:
DPRO-XXXX-XXXX-XXXX-XXXX-YYYYMMDD-XXXXXXXX
Where:
- First 5 segments: 4-character hex groups
- Date segment: Subscription expiration (YYYYMMDD)
- Final segment: 8-character cryptographic signature (SHA-256 truncated)

Validation Logic (Current Implementation):
1. Verify prefix equals "DPRO-"
2. Verify 6 segments separated by hyphens
3. Verify date segment is valid and not expired
4. Verify cryptographic signature matches key content
5. Check against revocation database (online check)

Note: The subscription_validator.cpp implements these checks in the validate_license() function, returning STATUS_VALID (0), STATUS_EXPIRED (1), STATUS_INVALID (2), or STATUS_REVOKED (3). For testing bypass scenarios, use keys from the "Invalid format" or "Expired" sections to confirm the validator is functioning correctly before modification. The current server time for testing is simulated as 2024-10-15 in the test environment.
